# KPI3 作業者別・日別実績ヒートマップ 改修計画

配置パス: `results_record_db/docs/kpi3_heatmap_change_plan.md`

## 1. 文書の位置づけ

本書は、`results_record_db` のKPI3画面へ作業者別・日別実績ヒートマップを追加するための改修仕様書である。

- 既存機能の仕様は `results_record_db/README.md` の「仕様」欄を正とする。
- 今回追加する表示仕様は、本書の「仕様」および「受入条件」を正とする。
- 本書の「解説」と「実装上の補足」は非規範とする。
- 両文書の仕様が矛盾する場合は実装を中断し、矛盾箇所を報告する。推測による補完は行わない。

本書は将来の改修候補を定義するものであり、KPI1ヒートマップのセミナー実演には含めない。

## 2. 目的

### 仕様

KPI3タブに、既存の作業者別・日別実績棒グラフを残したまま、同じ集計結果を使用したヒートマップを追加する。

ヒートマップにより、選択した工程について、作業者ごとの日別完了台数を日付方向に一覧で確認できるようにする。

### 解説

既存の棒グラフは、選択した工程における作業者間の実績比較に向いている。

一方、表示期間が複数日に及ぶ場合、どの作業者がどの日に多く担当したか、未実績の日が継続していないかを一覧で追いにくい。作業者と日付を縦横に配置したヒートマップを追加することで、日別の偏りや担当状況を確認しやすくする。

今回の改修ではKPI3の定義、工程・作業者・期間フィルタおよび日別件数の計算方法を変更しない。既存集計結果の別の見せ方を仕様として追加する。

## 3. 対象

### 仕様

主な変更対象は次のファイルとする。

- `results_record_db/src/streamlit_app.py`
- `results_record_db/tests/test_kpi.py`

新しい外部ライブラリは追加しない。既存のAltair、Pandas、Streamlitを使用する。

## 4. KPI3集計仕様

### 仕様

ヒートマップは、既存の `build_kpi3()` が返すDataFrameを使用する。

作業実績を元の `work_log` から再集計してはならない。

既存集計の条件は次のとおりとする。

| 項目 | 仕様 |
|---|---|
| 集計対象 | 画面で選択された期間・工程・作業者に一致する `work_log` |
| 完了日 | `end_ts` から作成した `work_date` |
| 工程 | KPI3タブ内の既存工程選択で選択された1工程 |
| 作業者 | 画面上部で選択された作業者 |
| セルの値 | `work_count` |
| 日付の粒度 | 1日 |
| 作業者順 | 作業者名の昇順 |
| 日付順 | 選択期間の開始日から終了日まで昇順 |
| 未実績セル | 0として表示 |

ヒートマップ用のDataFrameには、少なくとも次の列を持たせる。

| 列 | 内容 |
|---|---|
| `work_date` | 完了日 |
| `date_label` | 画面表示用の日付 |
| `process_name` | 選択した工程 |
| `worker_name` | 作業者 |
| `work_count` | 日別完了台数 |

表示用DataFrameは、選択期間内の全日付と、画面上部で選択された全作業者の組合せを作り、既存の `build_kpi3()` の結果を結合して作成する。

既存集計に存在しない日付・作業者の組合せは `work_count=0` とする。

表示用DataFrameの作成では、日付・作業者のグリッド作成、0補完および並べ替えだけを行う。作業件数の再集計は行わない。

### 解説

例えば、ある作業者が選択期間中に実績を持っていても、選択した工程では特定日に作業していない場合、そのセルを空白ではなく0として表示する。

これにより、「データが欠けた」のか「その日は当該工程の実績がなかった」のかを画面上で区別しやすくする。

## 5. ヒートマップ表示仕様

### 仕様

ヒートマップは、KPI3タブ内の既存棒グラフの直後、CSVダウンロードボタンおよびデータ表の前に表示する。

ヒートマップは、既存のKPI3工程選択で選択された1工程を対象とする。ヒートマップ専用の工程選択部品は追加しない。

表示内容は次のとおりとする。

| 表示要素 | 仕様 |
|---|---|
| グラフタイトル | `作業者別・日別完了台数ヒートマップ（{selected_kpi3_process}）`。`{selected_kpi3_process}` は選択した工程名へ置換する |
| 横軸 | `date_label` |
| 横軸タイトル | `完了日` |
| 横軸順 | 選択期間の開始日から終了日まで昇順 |
| 縦軸 | `worker_name` |
| 縦軸タイトル | `作業者` |
| 縦軸順 | 作業者名の昇順 |
| セルの値 | `work_count` |
| 色 | 完了台数が多いほど濃くする |
| 色スケール | 最小値0、色範囲 `#f7fbff` → `#6baed6` の連続色 |
| 凡例タイトル | `完了台数` |
| ツールチップ | 完了日、工程、作業者、完了台数 |

セル内への数値表示は行わない。複数日・複数作業者を同時に表示した際の可読性を優先し、正確な件数はツールチップと既存データ表で確認できるようにする。

横軸の日付ラベルは、日別セルを維持したまま、重なりを避けて読める表示とする。ラベルの間引き、`MM/DD` 形式への短縮または角度調整は許容するが、週単位や月単位へ再集計してはならない。

ヒートマップの高さは、作業者数に応じて各行が判別できるようにする。極端に高くなる場合は上限を設けてもよいが、作業者を無断で除外してはならない。

対象データがない場合はヒートマップを表示せず、例外を発生させない。

## 6. 既存機能の維持

### 仕様

次の既存機能は変更しない。

- KPI3の作業者別・日別実績棒グラフ
- KPI3の工程選択
- KPI3のCSVダウンロード
- KPI3の集計結果表
- 期間・工程・作業者フィルタ
- KPI3の集計方法
- Importタブ
- KPI1工程別時間別
- KPI2工程間滞留
- DB接続、データ取込、正規化およびreject処理

棒グラフ、ヒートマップ、KPI3 CSVおよび集計結果表は、同じ `build_kpi3()` の結果を基準とする。

ヒートマップの0補完は表示上の補完であり、KPI3 CSVおよび既存集計結果表へ0件行を追加してはならない。

## 7. 対象外

### 仕様

今回の改修では次を行わない。

- KPI3の計算式または日別粒度の変更
- 棒グラフからヒートマップへの置換
- 作業時間、標準時間、効率または生産性指標の追加
- 作業者の順位付けまたは評価
- 目標値、警告値、良否判定または閾値の追加
- 作業者の技能、所属、シフトまたは勤務状態との結合
- 複数工程を合算したセルの作成
- 週単位または月単位への再集計
- DBスキーマまたはSQLの変更
- ファイル構成の再編
- KPI3以外の画面デザイン変更
- 新しい外部ライブラリの追加
- 今回の表示追加と無関係なリファクタリング

## 8. 受入条件

以下をすべて満たした場合に改修完了とする。

1. KPI3タブに既存棒グラフが残っている。
2. 既存棒グラフの直後にヒートマップが表示される。
3. 既存の工程選択が棒グラフとヒートマップの両方へ反映される。
4. 横軸に選択期間内の日付が昇順で配置される。
5. 縦軸に画面上部で選択された作業者が昇順で配置される。
6. 画面上部で作業者を絞ると、ヒートマップの行も同じ作業者へ絞られる。
7. 完了台数が多いセルほど色が濃く表示される。
8. 実績のない日付・作業者のセルが0件として最小色で表示される。
9. ツールチップで完了日、工程、作業者、完了台数を確認できる。
10. ヒートマップの0件以外のセル値が、同じ工程・日付・作業者のKPI3集計結果と一致する。
11. 日別件数を週単位または月単位へ再集計していない。
12. KPI3 CSVの列、ファイル名およびダウンロード機能が変わっていない。
13. KPI3 CSVおよび既存集計結果表へ、ヒートマップ表示用の0件行が追加されていない。
14. Import、KPI1、KPI2の動作が変わっていない。
15. 対象データがない場合に例外が発生しない。
16. 新しい依存ライブラリが追加されていない。
17. `python -m pytest -q` が成功する。

## 9. テスト仕様

### 自動テスト

既存テストを維持し、少なくとも次を確認するテストを `results_record_db/tests/test_kpi.py` に追加する。

- 3日間・作業者2名を選択した場合、3日 × 2名 = 6セル分の表示用データが生成される。
- 選択した工程以外の集計行が含まれない。
- 選択していない作業者の行が含まれない。
- 実績のない日付・作業者の `work_count` が0になる。
- 0件以外のセル値が、元のKPI3 DataFrameの同一工程・日付・作業者と一致する。
- 日付が選択期間の開始日から終了日まで昇順になる。
- 作業者が作業者名の昇順になる。
- 空のDataFrameを渡した場合に空の表示用DataFrameが返り、例外が発生しない。
- 既存の期間・工程・作業者フィルタによるKPI3集計結果が維持される。
- ヒートマップ表示用の0件行が、KPI3 CSV用DataFrameへ混入しない。

チャート定義を関数として分離する場合は、Altairへ渡すフィールド、タイトル、色スケールおよびツールチップが仕様どおりであることも確認する。

### 手動確認

1. `results_record_db` へ移動する。
2. `python -m pytest -q` を実行する。
3. `python -m streamlit run src/streamlit_app.py` で画面を起動する。
4. KPI3タブを開く。
5. 既存棒グラフが維持されていることを確認する。
6. KPI3の工程を切り替え、棒グラフとヒートマップが同じ工程へ切り替わることを確認する。
7. 期間を1日に絞り、ヒートマップが1列になることを確認する。
8. 期間を複数日に広げ、日付ごとの列が増えることを確認する。
9. 作業者を1名に絞り、ヒートマップも1行になることを確認する。
10. 0件セルが最小色で表示されることを確認する。
11. 色の濃いセルへマウスを合わせ、棒グラフおよびKPI3集計結果表と同じ件数が表示されることを確認する。

## 10. 実装上の補足

本節は非規範とする。

- `pd.date_range(start_date, end_date, freq="D")` と選択作業者の直積で表示用グリッドを作成すると実装しやすい。
- 既存の工程選択後の `view_df` をグリッドへ左結合し、欠損した `work_count` だけを0で補完できる。
- Altairの `mark_rect()` を使用し、`date_label` と `worker_name` を名義尺度として扱うと順序を固定しやすい。
- 日付ラベルは表示専用とし、並び順には元の `work_date` を使用する。
- セル内数値は表示せず、既存の表とツールチップを正確な値の確認手段とする。
- ヒートマップ用の0補完済みDataFrameを、CSVダウンロードや既存データ表へ渡さない。
- 変更差分を小さく保つため、KPI3タブの描画処理と必要なテスト以外は変更しない。
