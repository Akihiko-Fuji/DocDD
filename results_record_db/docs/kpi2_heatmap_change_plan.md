# KPI2 工程間仕掛りヒートマップ 改修計画

配置パス: `results_record_db/docs/kpi2_heatmap_change_plan.md`

## 1. 文書の位置づけ

本書は、`results_record_db` のKPI2画面へ工程間仕掛りヒートマップを追加するための改修仕様書である。

- 既存機能の仕様は `results_record_db/README.md` の「仕様」欄を正とする。
- 今回追加する表示仕様は、本書の「仕様」および「受入条件」を正とする。
- 本書の「解説」と「実装上の補足」は非規範とする。
- 両文書の仕様が矛盾する場合は実装を中断し、矛盾箇所を報告する。推測による補完は行わない。

本書は将来の改修候補を定義するものであり、KPI1ヒートマップのセミナー実演には含めない。

## 2. 目的

### 仕様

KPI2タブに、既存の工程間仕掛り推移折れ線グラフを残したまま、同じ15分足の集計結果を使用したヒートマップを追加する。

ヒートマップにより、選択した工程間について、仕掛りが集中した日付と時間帯を色の濃さで確認できるようにする。

### 解説

既存の折れ線グラフは、時間経過に伴う仕掛り件数の増減や、2つの工程間の比較に向いている。

一方、複数日を含む期間では、日ごとの同じ時間帯に仕掛りが繰り返し発生しているかを読み取りにくい。日付と時刻を縦横に配置したヒートマップを追加することで、特定の曜日や時間帯に仕掛りが集中する傾向を確認しやすくする。

今回の改修ではKPI2の定義、工程間仕掛りの判定、15分足の計算方法を変更しない。既存集計結果の別の見せ方を仕様として追加する。

## 3. 対象

### 仕様

主な変更対象は次のファイルとする。

- `results_record_db/src/streamlit_app.py`
- `results_record_db/tests/test_kpi.py`

新しい外部ライブラリは追加しない。既存のAltair、Pandas、Streamlitを使用する。

## 4. KPI2集計仕様

### 仕様

ヒートマップは、既存の `build_kpi2()` が返す `trend` DataFrameを使用する。

工程間仕掛りの件数を `detail` DataFrameから再計算してはならない。

既存集計の条件は次のとおりとする。

| 項目 | 仕様 |
|---|---|
| 集計対象 | 画面で選択された期間に関係する `work_log` |
| 工程間 | 内装組立 → 外装組立、外装組立 → 出荷検査 |
| 仕掛り開始 | 前工程の `end_ts` |
| 仕掛り終了 | 次工程の `start_ts` |
| 集計間隔 | 15分 |
| 表示時間 | 08:00～16:45 |
| セルの値 | 各15分時点の `stagnation_count` |
| 日付 | `timestamp` から取得した日付 |
| 時刻 | `timestamp` から取得した15分単位の時刻 |

セルの値は、その15分間に発生した件数ではなく、その時点で次工程が未着手である仕掛り件数を示す。

ヒートマップ用のDataFrameには、少なくとも次の列を持たせる。

| 列 | 内容 |
|---|---|
| `pair` | 工程間 |
| `work_date` | `timestamp` の日付 |
| `time_slot` | `timestamp` の時刻を `HH:MM` 形式にした値 |
| `time_order` | 08:00を0として15分ごとに増加する並び順 |
| `stagnation_count` | 当該時点の仕掛り件数 |

表示用DataFrameの作成では、日付・時刻・工程間の並べ替えおよび不足セルの0補完だけを行う。KPI2の判定ロジックや件数の再集計は行わない。

### 解説

例えば、内装組立が08:10に完了し、外装組立が09:00に開始した場合、既存KPI2のルールに従い、該当する15分足の時点で仕掛りとして表示される。

ヒートマップはこの結果を日付と時刻へ分解して並べるだけであり、仕掛りの開始・終了判定には関与しない。

## 5. ヒートマップ表示仕様

### 仕様

ヒートマップは、KPI2タブ内の既存折れ線グラフの直後、CSVダウンロードボタンおよびデータ表の前に表示する。

既存折れ線グラフでは2つの工程間を同時に表示する。ヒートマップでは、表示密度を抑えるため、工程間を1つ選択して表示する。

工程間選択にはStreamlitの選択部品を使用する。

| 項目 | 仕様 |
|---|---|
| 選択ラベル | `ヒートマップに表示する工程間` |
| 選択肢1 | `内装組立 → 外装組立` |
| 選択肢2 | `外装組立 → 出荷検査` |
| 初期値 | `内装組立 → 外装組立` |

ヒートマップの表示内容は次のとおりとする。

| 表示要素 | 仕様 |
|---|---|
| グラフタイトル | `工程間仕掛りヒートマップ（{selected_pair}）`。`{selected_pair}` は選択した工程間名へ置換する |
| 横軸 | `time_slot` |
| 横軸タイトル | `時刻（15分単位）` |
| 横軸順 | 08:00から16:45まで昇順 |
| 縦軸 | `work_date` |
| 縦軸タイトル | `日付` |
| 縦軸順 | 選択期間の開始日から終了日まで昇順 |
| セルの値 | `stagnation_count` |
| 色 | 仕掛り件数が多いほど濃くする |
| 色スケール | 最小値0、色範囲 `#f7fbff` → `#6baed6` の連続色 |
| 凡例タイトル | `仕掛り件数` |
| ツールチップ | 日付、時刻、工程間、仕掛り件数 |

セル内への数値表示は行わない。15分足と複数日を同時に表示した際の可読性を優先し、正確な件数はツールチップと既存データ表で確認できるようにする。

横軸の時刻ラベルは、15分単位のセルを維持したまま、重なりを避けて読める表示とする。ラベルの間引きまたは角度調整は許容するが、セル自体を1時間単位へ再集計してはならない。

対象データがない場合はヒートマップを表示せず、例外を発生させない。

## 6. 既存機能の維持

### 仕様

次の既存機能は変更しない。

- KPI2の工程間仕掛り推移折れ線グラフ
- KPI2推移CSVダウンロード
- KPI2明細CSVダウンロード
- KPI2推移データ表
- KPI2明細データ表
- 工程間仕掛りの判定方法
- 15分足の集計方法
- 不正な工程順を明細へ残し、仕掛り件数から除外する処理
- 期間フィルタ
- Importタブ
- KPI1工程別時間別
- KPI3作業者別日別実績
- DB接続、データ取込、正規化およびreject処理

折れ線グラフ、ヒートマップ、KPI2推移CSVおよび推移データ表は、同じ `build_kpi2()` の `trend` DataFrameを基準とする。

## 7. 対象外

### 仕様

今回の改修では次を行わない。

- KPI2の計算式または15分足の変更
- 折れ線グラフからヒートマップへの置換
- 仕掛り時間の平均値、中央値または最大値の追加
- 目標値、警告値、良否判定または閾値の追加
- ネック工程または異常の自動判定
- 稼働日、休日、シフトまたは休憩時間の判定追加
- 2つの工程間を合算したセルの作成
- DBスキーマまたはSQLの変更
- ファイル構成の再編
- KPI2以外の画面デザイン変更
- 新しい外部ライブラリの追加
- 今回の表示追加と無関係なリファクタリング

## 8. 受入条件

以下をすべて満たした場合に改修完了とする。

1. KPI2タブに既存折れ線グラフが残っている。
2. 既存折れ線グラフの直後に工程間選択部品とヒートマップが表示される。
3. 工程間選択の初期値が `内装組立 → 外装組立` である。
4. 工程間を切り替えると、選択した工程間だけがヒートマップへ表示される。
5. 横軸に08:00～16:45の15分足が昇順で配置される。
6. 縦軸に選択期間内の日付が昇順で配置される。
7. 仕掛り件数が多いセルほど色が濃く表示される。
8. 0件のセルが色スケールの最小色で表示される。
9. ツールチップで日付、時刻、工程間、仕掛り件数を確認できる。
10. ヒートマップのセル値が、同じ `pair` と `timestamp` のKPI2推移データと一致する。
11. 15分足を1時間単位などへ再集計していない。
12. KPI2推移CSVとKPI2明細CSVの列、ファイル名およびダウンロード機能が変わっていない。
13. 不正な工程順のレコードが仕掛り件数へ加算されない既存動作が維持される。
14. Import、KPI1、KPI3の動作が変わっていない。
15. 対象データがない場合に例外が発生しない。
16. 新しい依存ライブラリが追加されていない。
17. `python -m pytest -q` が成功する。

## 9. テスト仕様

### 自動テスト

既存テストを維持し、少なくとも次を確認するテストを `results_record_db/tests/test_kpi.py` に追加する。

- 1日分の表示用データに、08:00～16:45の15分足が36セル生成される。
- 2日分では、1工程間につき36セル × 2日 = 72セル生成される。
- `timestamp` から `work_date`、`time_slot`、`time_order` が正しく作られる。
- 最初の時刻が08:00、最後の時刻が16:45になる。
- 選択していない工程間の行が含まれない。
- 実績のない日付・時刻の `stagnation_count` が0になる。
- 表示用データの件数が、元の `trend` DataFrameの同一 `pair`・`timestamp` と一致する。
- 空のDataFrameを渡した場合に空の表示用DataFrameが返り、例外が発生しない。
- 既存のKPI2仕掛り判定テストがすべて成功する。

チャート定義を関数として分離する場合は、Altairへ渡すフィールド、タイトル、色スケールおよびツールチップが仕様どおりであることも確認する。

### 手動確認

1. `results_record_db` へ移動する。
2. `python -m pytest -q` を実行する。
3. `python -m streamlit run src/streamlit_app.py` で画面を起動する。
4. KPI2タブを開く。
5. 既存折れ線グラフが維持されていることを確認する。
6. 工程間を切り替え、ヒートマップの内容が切り替わることを確認する。
7. 期間を1日に絞り、08:00～16:45の15分足が1行で表示されることを確認する。
8. 期間を複数日に広げ、日付ごとの行が増えることを確認する。
9. 色の濃いセルへマウスを合わせ、折れ線グラフおよび推移データ表と同じ件数が表示されることを確認する。
10. 0件のセルが最小色で表示されることを確認する。

## 10. 実装上の補足

本節は非規範とする。

- `trend["timestamp"]` から `work_date` と `time_slot` を作成すると実装しやすい。
- `time_order` は `timestamp.hour * 60 + timestamp.minute` を基準にすると並び順を固定しやすい。
- 工程間の選択後にDataFrameを絞り、Altairの `mark_rect()` で表示できる。
- 縦軸の日付は文字列化して名義尺度として扱うと、日付ごとの行を固定しやすい。
- セル内数値は表示せず、既存の表とツールチップを正確な値の確認手段とする。
- ヒートマップ用処理は表示用データの整形に限定し、`build_kpi2()` の仕掛り判定へ組み込まない。
- 変更差分を小さく保つため、KPI2タブの描画処理と必要なテスト以外は変更しない。
