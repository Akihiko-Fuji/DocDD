# Security Policy

## 対象範囲

本リポジトリはセミナー・学習用途のサンプルです。業務本番で必要となる認証、認可、秘密情報管理、通信暗号化、監査、バックアップ、可用性設計をすべて備えた製品ではありません。

セキュリティ修正は原則として最新の `main` ブランチを対象にします。過去コミットや教材の派生版に対する個別サポートは行いません。

## 脆弱性の報告

脆弱性、認証情報、個人情報など機微な内容を公開Issueへ投稿しないでください。GitHubリポジトリの **Security → Report a vulnerability** が利用できる場合は、非公開の脆弱性報告を使用してください。

非公開報告が利用できない場合は、公開Issueには詳細を書かず、「非公開で連絡したいセキュリティ上の懸念がある」ことだけを連絡してください。

報告には、可能な範囲で次を含めてください。

- 影響するファイルまたは機能
- 再現条件
- 想定される影響
- 回避策または修正案

## 教材用認証情報

`results_user` / `results_pass` は、ローカルPostgreSQL教材を短時間で動かすための公開済みデモ値です。秘密情報ではありません。共有環境、社内サーバー、クラウド環境、本番環境では使用しないでください。
