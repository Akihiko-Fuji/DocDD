# 前提条件と制約 / Assumptions and Constraints

## 1. 前提

### 1.1 文書解釈の前提
- 文書は人間と AI の双方が読み、理解し、妥当性評価に使うことを前提とする
- 文書だけを読んでも役割と責務が判断しやすい粒度を保つ
- 曖昧な指示や未確定事項は、放置せずに未決事項か補完方針を明示する

### 1.2 題材と実装環境の前提
- 題材は公開可能な落下ブロックゲームとする
- 実装対象は PC とする
- 画像・キャラクタ素材は独自制作とする
- 操作体系は Game Boy 版準拠の論理入力を PC へ写像する
- 想定描画解像度は 640×480 とし、PC 利用を前提とした固定基準解像度とする

## 2. 制約

### 2.1 参照元補完の制約
- 指示が不明瞭な場合は、Game Boy 版テトリスを主要参照元のひとつとして補完してよい
- ただし補完内容は、対応する要求・仕様・設計・記録文書へ反映し、暗黙仕様にしない

### 2.2 プロダクト制約
- 10×18 盤面を固定とする
- NEXT は 1 個のみとする
- Hold と Hard drop は採用しない
- A-TYPE を初期開発主軸とする
- B-TYPE は参考仕様とする
- T-Spin は独自拡張として採用する

## 3. 技術制約
- コードはコンパクトで見通し良く保つ
- 過剰なフレームワーク依存を避ける
