# 課題管理 / Issue Management

- 文書ID: DOC-MGT-054
- 最終更新日: 2026-03-23

## 1. 目的
未解決事項、実装着手前に決めるべき事項、将来拡張へ送る事項を分けて管理する。

## 2. 現在の主要論点
- 実装言語選定の最終確定
- replay 永続化の実装タイミング
- B-TYPE 参考仕様をどのフェーズで具体化するか

## 3. 解決済み論点
- ピース出現アルゴリズム方針は `ADR-0002` で固定
- 入力バッファ方針は `ADR-0003` で固定
- ゲームループ方針は `ADR-0001` で固定

## 4. 管理ルール
- コア10文書へ影響する課題は優先して上流文書へ反映する
- 設計判断へ格上げすべき課題は ADR または Decision Log へ移す
- 将来拡張へ送る場合でも、現時点での非対象理由を残す
