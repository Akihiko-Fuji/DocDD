# WBS / Work Breakdown Structure

- 文書ID: DOC-MGT-051
- 最終更新日: 2026-03-23

## 1. 目的
DocDD 見本として、文書更新から実装準備までの主要作業分解を示す。

## 2. 作業分解
1. 上位文書更新
   - project charter / scope / vision の見直し
2. 外部仕様更新
   - game rules / UI / input / scoring / rotation の同期
3. 内部設計更新
   - state machine / data model / module / diagram の同期
4. テスト文書更新
   - strategy / TC / traceability / DoD の同期
5. 記録更新
   - decision / change / review log の反映
6. 実装準備
   - ADR 固定事項の確認
   - schema / sample の確認

## 3. 受入観点
- 文書更新順が upstream -> downstream になっていること
- 変更時に records 更新が WBS に含まれていること
