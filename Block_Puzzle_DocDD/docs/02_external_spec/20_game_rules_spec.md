## 1. 本書の目的

本書は、プレイヤーから見たゲーム進行ルールを定義する。主対象は**Game Boy 版テトリスをベースラインとした A-TYPE 相当モード**であり、B-TYPE は参照仕様として扱う。T-Spin は本プロジェクト独自拡張である。

---

## 2. 仕様方針

1. 1 人用 A-TYPE を主対象とする
2. 盤面は 10×18 の可視領域とする
3. NEXT は 1 個、Hold はなしとする
4. SELECT で NEXT 表示を切り替えられる
5. Hard drop は採用しない
6. Game Boy 系の簡潔さを保ちながら、T-Spin を独自拡張として採用する
7. B-TYPE の 25 lines / HIGH は参照仕様として保持する
8. modern guideline 系 7-bag は採用しない

---

## 3. モード定義

### 3.1 A-TYPE（主対象）

- エンドレスモードとする
- プレイヤーは開始レベル 0〜9 を選択する
- ライン消去に応じてレベルが上昇する
- ゲームオーバーになるまでプレイを継続する

### 3.2 B-TYPE（参照仕様）

- 25 lines の達成を目指すモードとする
- 開始レベル 0〜9、HIGH 0〜5 を設定できる
- HIGH の値に応じて初期盤面へランダムブロックを積む
- プレイ中のレベル変動は行わない

### 3.3 対象外

- 2 人対戦の初期実装
- 通信モード
- ストーリー / ミッション系

---

## 4. 基本構成要素

### 4.1 盤面

主盤面は 10 列 × 18 行の可視領域とする。

### 4.2 ピース

I, O, T, J, L, S, Z の 7 種テトロミノを用いる。

### 4.3 操作対象

プレイヤーが直接操作できる落下中ピースは常に 1 個のみとする。

### 4.4 NEXT

- 次に出現するピースを 1 個先まで管理する
- プレイ画面では NEXT 表示を ON/OFF 切替できる
- 表示 OFF の場合も内部の次ピース管理は継続する

### 4.5 非採用機能

- Hold
- Hard drop
- ゴースト表示
- Combo
- Back-to-Back
- Perfect Clear ボーナス

---

## 5. 基本操作

| 入力 | 役割 |
|---|---|
| Left | 左へ 1 マス移動 |
| Right | 右へ 1 マス移動 |
| Down | ソフトドロップ |
| A | 90 度時計回り回転 |
| B | 90 度反時計回り回転 |
| SELECT | NEXT 表示 ON/OFF 切替 |
| START | 開始 / 一時停止 / 再開 |

入力詳細は `22_input_operation_spec.md` に委譲する。

---

## 6. 新規ゲーム開始時の初期状態

- 盤面を空にする
- スコアを 0 にする
- 累計ライン数を 0 にする
- レベルを開始レベルに設定する
- NEXT 表示状態を既定 ON にする
- 次ピースを 1 個生成する
- 現在ピースを出現させる
- 状態をプレイ中へ遷移させる

### 6.1 A-TYPE の開始条件

開始レベルは 0〜9 の範囲で選択できる。

### 6.2 B-TYPE の開始条件（参照）

開始レベル 0〜9、HIGH 0〜5 を設定できる。

### 6.3 既定値

開始レベル未選択時は 0 を既定値としてよい。

---

## 7. ピース出現ルール

### 7.1 出現位置

- 出現位置の基準は Game Boy 系の中央上部出現に準拠する
- 出現姿勢はピース種ごとに固定する
- 同種ピースは毎回同じ出現姿勢で出現する
- 出現位置の厳密座標は `24a_piece_shape_spawn_spec.md` を正本とする

### 7.2 出現手順

1. 現在ピース固定後、完成ライン判定と消去を行う
2. 得点、ライン数、レベルを更新する
3. NEXT を現在ピースとして出現させる
4. 新しい NEXT を補充する
5. 既定の出現姿勢・座標で置けない場合はゲームオーバーとする

### 7.3 randomizer 方針

- 出現列は 7-bag を保証しない
- Game Boy 系の偏りを持つ疑似乱数列を許容する
- 同一 seed と同一入力列では同一出現列を再現できること

---

## 8. 落下・固定・タイミング

### 8.1 自動落下

現在ピースは、現在レベルに応じた速度で自動落下する。速度表は `23_scoring_level_spec.md` を正本とする。

### 8.2 ソフトドロップ

- Down 入力中は通常落下より速い落下を行う
- Game Boy 基準として外部上は 1/3G 相当の高速落下を採用する
- ソフトドロップ中も左右移動と回転は可能とする

### 8.3 ARE / ライン消去待ち

- ピース出現直後には短い出現待ち（ARE）が存在する
- ライン消去時には視認可能な待ち時間を持つ
- 具体値（ARE / ライン消去待ち / ソフトドロップ）は `23a_timing_constants_spec.md` を正本とする

### 8.4 接地

現在ピースが下方向へ進めない状態を接地とみなす。

### 8.5 固定

接地して下方向移動不能となった現在ピースは固定対象となる。

### 8.6 固定後処理順序

1. 完成ライン判定
2. ライン消去
3. T-Spin 判定確定
4. 得点更新
5. ライン数更新
6. レベル更新
7. 次ピース準備
8. 出現可否判定

---

## 9. 移動・回転・衝突

- 左右移動および回転は、結果位置が有効位置である場合のみ成立する
- 盤面外、固定済みブロック重なりは不成立とする
- wall kick / floor kick は採用しない
- 詳細は `24_piece_rotation_collision_spec.md` に委譲する

---

## 10. ライン消去ルール

### 10.1 完成条件

横 1 列が左端から右端まで埋まった場合、その列を完成ラインとする。

### 10.2 消去単位

同一タイミングで 1〜4 行を同時消去できる。

### 10.3 詰め処理

消去行より上のブロック群を行単位で下へ平行移動する。

### 10.4 同時消去行数の確定

同時消去行数は 1, 2, 3, 4 のいずれかとして得点計算へ渡さなければならない。

---

## 11. 得点・レベル・速度の概要

### 11.1 得点方針

得点は以下の要素で構成する。正本は `23_scoring_level_spec.md` とする。

- 通常ライン消去得点
- T-Spin 得点
- ソフトドロップ得点

### 11.2 通常ライン消去基本点

| 同時消去行数 | 基本点 |
|---|---:|
| 1 | 40 |
| 2 | 100 |
| 3 | 300 |
| 4 | 1200 |

### 11.3 A-TYPE 加算式

`実加算点 = 基本点 × (レベル + 1)`

### 11.4 レベル進行

- 最初のレベル上昇条件は `開始レベル × 10 + 10` ラインとする
- 以後は 10 ラインごとに 1 レベル上昇する
- 上限は 20 とする

### 11.5 速度段階

- レベル 0〜20 に応じて 1 行あたりの落下フレーム数を定義する
- Level 0 は 53 frames/row、Level 9 は 11 frames/row、Level 20 は 3 frames/row を基準とする

### 11.6 T-Spin

T-Spin が成立した場合は通常ライン消去得点より T-Spin 専用得点を優先適用する。

---

## 12. 補助仕様

### 12.1 SELECT による NEXT 表示切替

- プレイ中に SELECT を押すと NEXT 表示を切り替える
- 表示 ON/OFF はプレイ利便性の補助であり、内部のピース順には影響しない
- 一時停止中やゲームオーバー中は切替対象外とする

### 12.2 ハートレベル（参照）

Game Boy 版には高速モードが存在するが、本プロジェクトでは初期実装対象外とし、参照情報としてのみ保持する。

### 12.3 リセットコマンド（参照）

Game Boy 版の A+B+SELECT+START 同時押しによるリセットは、初期実装対象外とする。

---

## 13. ゲームオーバー条件

- 新しい現在ピースを規定位置へ出現できない場合、ゲームオーバーとする
- ゲームオーバー成立後は通常プレイ入力を無効化する
- 最終 SCORE / LINES / LEVEL を固定表示する

---

## 14. Spec-Driven / Acceptance-Driven 観点

1. A-TYPE の開始からゲームオーバーまでのルール順序が一貫していること
2. SELECT による NEXT 表示切替が暗黙でなく明記されていること
3. B-TYPE の扱いが「参照仕様」として読めること
4. Game Boy ベースラインと T-Spin 拡張の境界が明確であること
5. 速度・得点・状態遷移が他文書へ追跡可能であること
