## 1. 本書の位置付け

本書は、プレイヤーが観測する代表フローを **要求と外部仕様の中間粒度** で整理する文書である。

- `13_functional_requirements.md` が「何を満たすべきか」を定義するのに対し、本書は「どの順で体験されるか」を定義する
- 詳細なルール値や回転条件は外部仕様へ委譲し、本書ではフロー上の判断点と分岐を明示する
- テストケース文書は、本書の基本・代替・例外フローを起点に派生させる

## 3. UC-001 A-TYPE を開始する

- アクター: プレイヤー
- 事前条件:
  - タイトル画面表示中である
  - 入力受付が有効である
- 事後条件:
  - `GameSession` が新規に初期化される
  - 開始レベルがセッションへ反映され、`ST-PLAY` へ遷移する

### 3.1 基本流れ
1. プレイヤーはタイトル画面で START 相当入力を行う
2. システムは A-TYPE 開始設定画面へ遷移する
3. プレイヤーは Left / Right で開始レベルを選択する
4. プレイヤーは START または A 相当入力で開始を確定する
5. システムは新規セッションを作成し、盤面・得点・NEXT を初期化する
6. システムはプレイ画面を表示し、最初のピース出現処理を開始する

### 3.2 代替フロー
- AF-001: レベル変更を行わず既定値で開始する
  1. プレイヤーはレベル変更を行わずに開始を確定する
  2. システムは `config.ini` または既定値から導いた開始レベルをそのまま適用する
- AF-002: 設定画面からタイトルへ戻る
  1. プレイヤーは B 相当入力で設定を中止する
  2. システムはセッションを作成せず、タイトル画面へ戻る

### 3.3 例外フロー
- EF-001: 開始レベル設定値が不正な場合
  1. システムは範囲外値を検出する
  2. システムは安全な既定値または有効範囲内の値へ正規化して開始する
- EF-002: 新規セッション初期化に必要な設定読込に失敗した場合
  1. システムは設定読込失敗を検出する
  2. システムは既定設定へフォールバックし、プレイ開始可能なら開始を継続する
  3. 継続不能な内部異常であればタイトルに留まり、異常を検出可能にする

## 4. UC-002 プレイ中にピースを操作する

- アクター: プレイヤー
- 事前条件:
  - `ST-PLAY` 状態である
  - 現在ピースが盤面上に存在する
- 事後条件:
  - 現在ピース位置、盤面、得点、ライン数、レベルのいずれかが更新されるか、入力が無効として却下される

### 4.1 基本流れ
1. プレイヤーは盤面状況を見て左右移動、回転、Down によるソフトドロップを入力する
2. システムはフレーム単位で入力を `InputSnapshot` に正規化する
3. システムは有効入力のみを優先順に評価する
4. システムは移動または回転が成立した場合、現在ピース状態を更新する
5. システムは下移動不能を検出した場合、固定処理へ進む
6. システムはライン消去、T-Spin 判定、得点更新、レベル更新を順に適用する
7. システムは次ピースを準備し、継続可能なら次フレームの操作受付へ戻る

### 4.2 代替フロー
- AF-201: 回転が衝突または盤面外で失敗する
  1. プレイヤーは A または B を押す
  2. システムは回転後位置が不正であることを検出する
  3. システムは current piece を更新せず、元の向きと位置を維持する
  4. システムは最終成立操作を回転以外のまま保持する
- AF-202: `Left + Right` 同時押下が発生する
  1. システムは左右同時押下を検出する
  2. システムは左右移動を相殺し、他の優先入力のみ評価する
- AF-203: `A + B` 同時押下が発生する
  1. システムは両回転入力を検出する
  2. システムは回転を不成立として扱い、以降の左右移動・Down・自動落下判定を継続する
- AF-204: T-Spin を狙ったが成立しない
  1. プレイヤーは T ピースを回転させて固定する
  2. システムは対角占有数または最終成立操作条件を評価する
  3. システムは T-Spin 不成立と判定し、通常のライン消去得点表または 0 点処理を適用する
- AF-205: ラインが消えずに固定のみが発生する
  1. システムは完成ラインなしを検出する
  2. システムは得点表に従って必要な加点のみ行い、次ピース準備へ進む

### 4.3 例外フロー
- EF-201: 出現直後に配置不能でゲームオーバーになる
  1. システムは新しい現在ピースの初期配置衝突を検出する
  2. システムは `ST-GAMEOVER` へ遷移する
- EF-202: 不正な入力集合が内部へ渡された場合
  1. システムは論理入力集合に未知ボタンまたは禁止入力を検出する
  2. システムはその入力を無効として扱い、必要に応じてログまたは assertion で検出可能にする
- EF-203: 得点更新や T-Spin 判定の内部整合性が崩れた場合
  1. システムは矛盾を検出する
  2. システムはクラッシュよりも検出可能性を優先し、ログまたは開発時 assertion により異常を表面化する

## 5. UC-003 一時停止する

- アクター: プレイヤー
- 事前条件:
  - `ST-PLAY` 状態である
- 事後条件:
  - `ST-PAUSE` へ遷移するか、再開・タイトル復帰が完了する

### 5.1 基本流れ
1. プレイヤーはプレイ中に START を押す
2. システムは同フレームで START を最優先入力として扱う
3. システムはプレイ進行を停止し、PAUSE 表示へ切り替える
4. プレイヤーは START で再開する、または B / ESC でタイトルへ戻る

### 5.2 代替フロー
- AF-301: START と他入力が同時に押される
  1. システムは START を優先し、他のプレイ入力を破棄する
  2. 盤面や現在ピースは pause entry 前の値を保持する
- AF-302: pause 中に通常プレイ入力が押される
  1. プレイヤーは Left / Right / Down / A / B を押す
  2. システムはタイトル復帰に割り当てられた B を除き、プレイ進行へ反映しない

### 5.3 例外フロー
- EF-301: 再開直後に同一押下継続で再 pause しそうになる
  1. システムは pause/resume 用バッファ方針に従い、再開直後 1 frame の START 再評価を抑止する
  2. システムは即時再 pause を発生させない
- EF-302: pause 中に状態遷移矛盾が検出される
  1. システムは `ST-PAUSE` で許可されない遷移要求を検出する
  2. システムはプレイ進行を再開せず、異常を記録可能にする

## 6. UC-004 ゲームオーバー後に再試行する

- アクター: プレイヤー
- 事前条件:
  - `ST-GAMEOVER` 状態である
- 事後条件:
  - 再試行なら `ST-SETUP-A` へ戻り、新規セッション開始準備が整う
  - タイトル復帰なら `ST-TITLE` へ戻る

### 6.1 基本流れ
1. プレイヤーはゲームオーバー画面を確認する
2. プレイヤーは再試行入力を行う
3. システムは前セッション結果を固定したまま開始設定画面へ戻る
4. プレイヤーは開始レベルを再設定し、新しいプレイを開始する

### 6.2 代替フロー
- AF-401: タイトルへ戻る
  1. プレイヤーはタイトル復帰入力を行う
  2. システムはセッションを破棄し、タイトルへ戻る
- AF-402: 再試行前に結果表示のみ確認する
  1. プレイヤーは即時入力せず結果画面に留まる
  2. システムは通常プレイ入力を受け付けず、再試行またはタイトル復帰入力のみ待機する

### 6.3 例外フロー
- EF-401: ゲームオーバー画面で通常プレイ入力が送られる
  1. システムは Left / Right / Down / SELECT 等の無効入力を受け取る
  2. システムはセッション再開や盤面更新を行わず、無効入力として破棄する
- EF-402: 再試行遷移前の再初期化に失敗する
  1. システムは新規セッション準備の内部異常を検出する
  2. システムは壊れたプレイ状態へ進まず、安全な開始設定画面またはタイトル画面へ留まる
