# テストフィクスチャ一覧 / Test Fixtures Catalog

- 文書ID: DOC-QA-046
- 最終更新日: 2026-03-23
- 目的: 盤面前提、入力列、開始状態を再利用可能な fixture として定義し、試験再現性を高める
- 関連文書:
  - `40_test_strategy.md`
  - `41_test_cases_game_rules.md`
  - `42_test_cases_ui_input.md`

## 1. 運用方針

- Fixture ID は文書上の正本 ID とし、将来ファイル化する場合も同じ ID を使う
- 盤面 fixture は ASCII 図、現在ピース、NEXT、開始状態を必ず持つ
- 入力 fixture はフレーム単位の入力列を持つ
- 1 つの fixture を複数 TC で再利用する場合、参照 TC を一覧化する

## 2. 記法

- `.` は空セル、`#` は固定済みセルを表す
- 行は上から下へ、列は左から右へ読む
- ASCII 図は可視盤面 10×18 を上 4 行省略可で記載する

## 6. UI / State Fixture 一覧

## 7. 受入観点

- 主要 TC が具体 fixture ID を参照できること
- 盤面、入力列、開始状態が再利用可能な粒度で定義されていること
- 回転失敗系 fixture でも ASCII 盤面、現在ピース、NEXT、開始状態が揃っていること
- 実装がなくても試験再現の前提が文書上で共有できること
