# 入力タイミング設計 / Input Timing Design

- 文書ID: DOC-DSN-036
- 最終更新日: 2026-03-23

## 1. 目的
入力受付とゲーム更新の関係を整理し、回転・移動・一時停止の解釈順序を明確化する。

## 2. 基本順序
1. 入力スナップショット取得
2. 状態依存で有効入力を抽出
3. 左右移動 / 回転 / ソフトドロップを評価
4. 自動落下を評価
5. 接地 / 固定判定へ渡す

## 3. START 優先
START はプレイ中に押された場合、同フレーム内で一時停止遷移を優先してよい。

## 4. T-Spin との関係
最終成立操作が回転かどうかを保持できるよう、成立した入力結果をフレーム単位で記録する。
