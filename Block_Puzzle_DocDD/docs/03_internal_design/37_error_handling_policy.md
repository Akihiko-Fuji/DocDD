## 1. 目的
想定内不整合や設定不備が発生した場合の扱いを定義し、進行不能や silent failure を防ぐ。

## 2. 方針
- 想定内不整合は既定値で継続可能なら継続する
- 状態遷移矛盾は早期に検出する
- T-Spin 判定や得点計算の矛盾はログ化しやすくする
- replay / config の読込不備ではクラッシュより既定値復帰を優先する

## 3. 想定内異常の例
- start_level が範囲外
- `config.ini` の入力設定に未知キーまたは未知ゲームパッドボタンが含まれる
- replay に未知ボタン名が含まれる
- 状態遷移不可能な画面で START 以外が送られる

## 4. 対応原則
1. config は既定値へ正規化する
2. replay は読み飛ばしではなく検出可能なエラーとして扱う
3. 状態遷移矛盾はログまたは assertion で検出しやすくする
4. UI には可能な限り破綻しない表示を残す

## 5. 受入観点
- NFR-301, NFR-302, NFR-303, NFR-304 と追跡できること
- 設定未整備でも最低限のプレイ開始が成立すること
- エラー時の責務分担が `34_module_design.md` と矛盾しないこと
