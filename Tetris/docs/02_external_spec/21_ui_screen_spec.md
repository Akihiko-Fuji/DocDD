## 1. 本書の目的

本書は、本プロジェクトにおける**プレイヤーから見える画面構成とその振る舞い**を定義するための文書である。  
本書では、以下を定義する。

- どの画面が存在するか
- 各画面で何を表示するか
- 各画面でどの入力を受け付けるか
- 画面間をどの条件で遷移するか

一方で、本書では以下は詳細定義しない。

- ピース回転や衝突判定の詳細
- ゲームループや内部状態更新順序
- 描画実装方式
- テストケース詳細
- 画像素材や演出アニメーションの制作仕様

これらは関連文書へ委譲する。

---

## 2. UI仕様の基本方針

本プロジェクトの UI は、**Game Boy版テトリス相当の簡潔な構成**を参照しつつ、PC向け実装として整理した独自仕様とする。  
本書における基本方針は以下の通りである。

1. 画面数は必要最小限に抑える
2. プレイに必要な情報を優先して表示する
3. 近代的な過剰UIは採用しない
4. 素材差し替え可能な前提で、表示要素と配置責務を明確にする
5. 画面仕様は、内部設計ではなく、**プレイヤーから見た見え方と操作可能性**に集中して記述する

## 4. 画面遷移概要

本プロジェクトの基本的な画面遷移は以下の通りとする。

### 4.1 初期実装対象の遷移
- SCR-001 タイトル画面
  - 新規開始操作 → SCR-002 ゲーム開始設定画面

- SCR-002 ゲーム開始設定画面
  - 開始確定 → SCR-003 プレイ画面
  - 戻る操作 → SCR-001 タイトル画面

- SCR-003 プレイ画面
  - START入力 → SCR-004 一時停止画面
  - ゲームオーバー条件成立 → SCR-005 ゲームオーバー / リザルト画面

- SCR-004 一時停止画面
  - 再開操作 → SCR-003 プレイ画面
  - タイトルへ戻る操作が採用される場合 → SCR-001 タイトル画面

- SCR-005 ゲームオーバー / リザルト画面
  - 再試行 → SCR-002 ゲーム開始設定画面
  - タイトルへ戻る → SCR-001 タイトル画面

### 4.2 将来拡張時の遷移
- SCR-001 タイトル画面
  - モード選択採用時 → SCR-006 モード選択画面

- SCR-006 モード選択画面
  - A-TYPE 選択 → SCR-002 ゲーム開始設定画面
  - B-TYPE 選択 → SCR-007 B-TYPE 設定画面

- SCR-007 B-TYPE 設定画面
  - 開始確定 → SCR-003 プレイ画面
  - 戻る操作 → SCR-006 モード選択画面

## 5. 共通UI方針

### 5.1 表示優先順位
表示優先順位は以下の通りとする。

1. プレイ成立に必要な情報
2. 現在状態の認識に必要な情報
3. スコアや進行度の補助情報
4. 装飾情報

### 5.2 素材利用方針
キャラクタ画像や装飾画像は独自制作素材を使用できるものとする。  
ただし、素材が未投入の段階でも、画面仕様としては成立しなければならない。

### 5.3 文字表示方針
数値・状態・操作案内は、素材の有無にかかわらず、文字表示のみでも判読可能でなければならない。

### 5.4 画面密度方針
近代的な情報過多UIは避け、1画面内の情報密度は必要最小限に抑える。  
特にプレイ画面では、盤面視認性を最優先とする。

### 5.5 状態依存入力方針
各画面は、現在の状態に応じて有効な入力のみを受け付ける。  
詳細な入力解釈は `32_state_machine_design.md` に委譲するが、プレイヤー視点では「押しても反応しない理由」が大きく曖昧にならないことを前提とする。

## 6. SCR-001 タイトル画面

### 6.1 目的
ゲーム開始前の入口画面として機能する。  
プレイヤーに対して、ゲームを開始できることを明確に示す。

### 6.2 表示要素
タイトル画面には、少なくとも以下を表示しなければならない。

- ゲームタイトル
- 開始可能であることを示す表示
- 必要最小限の操作案内

### 6.3 表示例
表示内容の例を以下に示す。

- ゲームロゴまたはタイトル文字
- `PRESS START`
- `PRESS ENTER`
- `START GAME`

補足:
- 実際の文言は実装側で調整してよい
- キーボード前提の実装では、STARTに相当するPCキー表記へ置き換えてよい

### 6.4 入力受付
タイトル画面では、少なくとも以下の入力を受け付けなければならない。

- ゲーム開始に進む入力

### 6.5 遷移
- 開始入力 → SCR-002 ゲーム開始設定画面
- モード選択を採用する場合は → SCR-006 モード選択画面

### 6.6 非表示項目
タイトル画面では、以下は原則表示しない。

- プレイ中の盤面
- リアルタイムスコア
- 詳細設定一覧
- 開発者向け情報

---

## 7. SCR-002 ゲーム開始設定画面

### 7.1 目的
A-TYPE 相当モードの開始前に、プレイヤーが開始条件を設定するための画面である。

### 7.2 初期実装で扱う設定項目
初期実装では、少なくとも以下を扱う。

- 開始レベル

### 7.3 開始レベル仕様
開始レベルは 0～9 の範囲で選択できなければならない。

### 7.4 表示要素
ゲーム開始設定画面には、少なくとも以下を表示しなければならない。

- 画面名または設定対象が分かる表示
- 現在選択中の開始レベル
- 開始確定方法
- 戻る方法

### 7.5 入力受付
少なくとも以下の入力を受け付ける。

- 値変更入力
- 開始確定入力
- 戻る入力

### 7.6 遷移
- 開始確定 → SCR-003 プレイ画面
- 戻る → SCR-001 タイトル画面

### 7.7 表示方針
この画面では、設定対象は必要最小限に留める。  
初期段階では、過度なオプション一覧を表示しない。

---

## 8. SCR-003 プレイ画面

### 8.1 目的
ゲーム本編のプレイを行う中核画面である。  
プレイヤーは本画面上でピース操作、状況把握、進行確認を行う。

### 8.2 必須表示要素
プレイ画面には、少なくとも以下を表示しなければならない。

- 盤面
- 現在ピース
- 固定済みブロック
- 次ピース
- スコア
- ライン数
- レベル
- 必要に応じた状態表示

### 8.3 盤面表示
盤面は、幅10マス × 高さ18マスの可視領域として認識できなければならない。

### 8.4 現在ピース表示
落下中の現在ピースは、固定済みブロックと明確に区別可能でなければならない。

### 8.5 固定済みブロック表示
盤面上に積み上がった固定済みブロックは、現在ピースと識別できる表示でなければならない。

### 8.6 次ピース表示
次に出現するピースは、1個先まで表示しなければならない。

### 8.7 数値情報表示
以下の数値は、プレイヤーがプレイ中に確認できなければならない。

- SCORE
- LINES
- LEVEL

### 8.8 装飾表示
キャラクタ画像や装飾UIは表示してよい。  
ただし、以下を満たすこと。

- 盤面視認性を阻害しない
- スコアや次ピース等の主要情報を隠さない
- 素材未使用でも画面が成立する

### 8.9 入力受付
プレイ画面では、少なくとも以下の入力を受け付けなければならない。

- 左移動
- 右移動
- 回転
- ソフトドロップ
- 一時停止

### 8.10 非採用要素
プレイ画面では、初期段階では以下を表示しない。

- Hold 枠
- ゴーストピース
- コンボ表示
- Back-to-Back 表示
- T-Spin 表示
- リプレイ制御UI

### 8.11 プレイ画面の構成原則
プレイ画面では、**盤面を視覚的主役**とする。  
スコア、レベル、ライン、次ピースは補助情報として配置する。

### 8.12 レイアウト自由度
画面内の具体的配置は実装側に委ねてよいが、少なくとも以下を守ること。

- 盤面が最も視認しやすい位置と大きさであること
- 次ピースが独立表示で識別しやすいこと
- SCORE / LINES / LEVEL が相互に混同しないこと

---

## 9. SCR-004 一時停止画面

### 9.1 目的
プレイ中断状態をプレイヤーへ明示し、再開可能であることを示す。

### 9.2 表示方式
一時停止画面は、以下のいずれかで実現してよい。

- プレイ画面上へのオーバーレイ表示
- 専用画面への遷移表示

ただし、プレイヤーから見て「現在プレイ進行が停止している」ことが明確でなければならない。

### 9.3 必須表示要素
少なくとも以下を表示しなければならない。

- 一時停止中であることを示す表示
- 再開方法
- 必要に応じた戻る方法

### 9.4 入力受付
一時停止画面では、少なくとも以下を受け付ける。

- 再開入力
- タイトルへ戻る入力を採用する場合はその入力

### 9.5 遷移
- 再開 → SCR-003 プレイ画面
- タイトルへ戻る採用時 → SCR-001 タイトル画面

### 9.6 停止中の見え方
一時停止中は、盤面を見せ続けてもよい。  
ただし、プレイヤーが「まだ進行している」と誤認しない表示でなければならない。

---

## 10. SCR-005 ゲームオーバー / リザルト画面

### 10.1 目的
ゲーム終了を明示し、結果を確認させたうえで、再試行またはタイトル復帰へ導く。

### 10.2 必須表示要素
少なくとも以下を表示しなければならない。

- ゲームオーバーであることを示す表示
- 最終スコア
- 最終ライン数
- 最終レベル
- 再試行方法
- タイトルへ戻る方法

### 10.3 任意表示要素
必要に応じて、以下を表示してよい。

- プレイ中盤面の最終状態
- 開始レベル
- 簡易メッセージ
- キャラクタ演出

### 10.4 入力受付
少なくとも以下の入力を受け付ける。

- 再試行
- タイトルへ戻る

### 10.5 遷移
- 再試行 → SCR-002 ゲーム開始設定画面
- タイトルへ戻る → SCR-001 タイトル画面

### 10.6 表示方針
ゲームオーバー表示は、プレイヤーが通常プレイ状態と混同しないよう、明確に区切られていなければならない。

---

## 11. SCR-006 モード選択画面（予約 / 将来拡張）

### 11.1 目的
将来的に A-TYPE / B-TYPE など複数モードを選択可能にする場合のための予約画面である。

### 11.2 表示要素
将来採用時には、少なくとも以下を表示する。

- モード一覧
- 現在選択中モード
- 決定方法
- 戻る方法

### 11.3 遷移
- A-TYPE 選択 → SCR-002 ゲーム開始設定画面
- B-TYPE 選択 → SCR-007 B-TYPE 設定画面
- 戻る → SCR-001 タイトル画面

### 11.4 初期実装方針
初期実装では省略してよい。  
A-TYPE 固定で開始する場合は、SCR-001 から SCR-002 へ直接遷移する。

---

## 12. SCR-007 B-TYPE 設定画面（予約 / 将来拡張）

### 12.1 目的
B-TYPE 採用時に、開始レベルおよび HIGH を設定するための予約画面である。

### 12.2 設定項目
将来採用時には、少なくとも以下を扱う。

- 開始レベル: 0～9
- HIGH: 0～5

### 12.3 遷移
- 開始確定 → SCR-003 プレイ画面
- 戻る → SCR-006 モード選択画面

### 12.4 初期実装方針
初期実装では省略してよい。

## 13. プレイ画面の情報配置要件

### 13.1 必須情報群
以下は、プレイ成立のための必須情報群とする。

- 盤面
- 現在ピース
- 次ピース
- SCORE
- LINES
- LEVEL

### 13.2 推奨配置原則
以下を推奨する。

- 盤面は中央または主視認領域に配置する
- 次ピースは盤面近傍の独立領域に配置する
- SCORE / LINES / LEVEL は縦並びまたは明確な区切りで配置する
- 装飾素材は外周側へ寄せる

### 13.3 禁止に近い配置
以下は避けなければならない。

- 盤面の視界を装飾が遮る
- 数値情報が互いに重なって読めない
- 次ピースが盤面と混同される
- 現在ピースと固定済みブロックが同一視されやすい

---

## 14. 画面ごとの入力有効範囲

本書では詳細な入力状態管理までは定義しないが、画面ごとの有効入力範囲は以下を原則とする。

| 画面ID | 主な有効入力 |
|---|---|
| SCR-001 | 開始 |
| SCR-002 | 設定変更、決定、戻る |
| SCR-003 | 左右移動、回転、ソフトドロップ、一時停止 |
| SCR-004 | 再開、必要に応じてタイトルへ戻る |
| SCR-005 | 再試行、タイトルへ戻る |
| SCR-006 | 項目選択、決定、戻る |
| SCR-007 | 設定変更、決定、戻る |

補足:
- 無効入力は無視してよい
- ただし、同じ画面内で入力解釈が不安定にぶれないこと

## 15. 表示文言方針

### 15.1 基本文言
画面文言は、簡潔で誤解しにくいものとする。

### 15.2 言語方針
初期実装では、日本語または英語のいずれか一方で統一してよい。  
将来多言語化の余地を残してもよいが、初期必須ではない。

### 15.3 数値ラベル
数値表示には、原則として以下のような短いラベルを用いる。

- SCORE
- LINES
- LEVEL
- NEXT
- PAUSE
- GAME OVER

### 15.4 実装依存表記
PC向け実装では、必要に応じて以下のような表記置換を行ってよい。

- START → ENTER
- START → P
- A/B → 任意のPCキー表記

ただし、プレイヤー向け表示と実際の操作定義が一致していなければならない。

## 16. 素材未投入時の成立条件

本プロジェクトでは独自素材を使用可能とするが、素材が未完成または未投入の段階でも、各画面は最低限成立しなければならない。

成立条件は以下の通りとする。

- 文字だけで画面役割が分かること
- プレイに必要な情報が表示されること
- 状態遷移が成立すること
- 入力可能内容が必要に応じて判別できること

## 18. 受入観点（本書レベル）

本書に基づく UI 仕様の受入観点は、少なくとも以下とする。

1. 初期実装に必要な画面が定義されていること
2. 各画面の目的、表示要素、入力、遷移が記載されていること
3. A-TYPE 主対象、B-TYPE 予約という境界が明確であること
4. プレイ画面で必要な情報が欠けていないこと
5. 内部実装や回転仕様の詳細を適切に下位文書へ委譲していること

## 19. 結論

本プロジェクトの UI は、**Game Boy版テトリス相当の簡潔さ**を参照しつつ、PC向けの DocDD 見本として整理する。

初期実装で中核となるのは以下である。

- タイトル画面
- ゲーム開始設定画面
- プレイ画面
- 一時停止画面
- ゲームオーバー / リザルト画面

UI の役割は、装飾ではなく、**プレイヤーが現在状態を理解し、必要な操作を迷わず行えるようにすること**にある。  
そのため本仕様では、盤面視認性、状態明確性、必要情報の簡潔表示を重視する。
