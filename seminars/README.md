## 今日の持ち帰り：仕様＝3文書

このセミナーの核心は、以下の3文書です。
プレゼン資料よりもこの3文書の方が、支援先で長く使われます。

仕様 = データ契約（01）＋ 受入条件（02）＋ 運用ガードレール（03）

| 文書 | 問いに答える | 対応するファイル |
|---|---|---|
| データ契約 | 何が「正しいデータ」か | `01_data_contract.md` |
| 受入条件 | どこまでできれば「完成」か | `02_acceptance.md` |
| 運用ガードレール | 誰がいつどう動かすか | `03_ops_guardrails.md` |

これが決まると「AIでも外注でもブレにくい」。
逆に、ここが曖昧だと、AIはそれっぽく埋めますが現場で破綻します。

## デモ構成
Excel/CSV（汚いデータ）
  ↓
PostgreSQL（UNIQUE制約で防衛）
  ↓
Streamlit（取込UI ＋ 集計 ＋ フィルタ ＋ 出力）
  ↓
不良報告（受注製番で接続）→ 改善会議で使える絵

サンプルデータ・DDL・取込コードは [`results_record_db/`](../../results_record_db/) を参照してください。

## 参加者向け：何を持ち帰るか

このセミナーが終わった時点で、以下ができるようになることを目標にしています。

1. 支援先の現場で「揉めている問題」を、3文書のどれに落とすか判断できる
2. データ契約の「同一判定キー」と「除外条件」を1つ書ける
3. 受入条件を「YES/NOで判定できる条件3つ」に落とせる
4. ベンダー・情シスへの依頼書（`04_vendor_brief.md`）の雛形を使って提案できる
